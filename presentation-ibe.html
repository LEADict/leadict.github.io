Cloud Computing - Security Concerns

Security vs. Privacy

  • Security - overall information protection
  • Privacy - individual information governance

  • Cloud Computing & Security - meet very well
  • Cloud Computing & Privacy - contradictory

Privacy is the issue!

  • Business information
  • Personal information

Encryption is one of the most effective data protection techniques.

  • Security - Data at Rest Encryption, Data in Transit Encryption
  • Privacy - Data in Use Encryption

Data Encryption & Privacy Preserving

Challenges:

  • Data Storage/Sharing & Privacy preserving
  • Cloud Computing technology integration
  • Decentralized Identity Management
  • Multi-trusted domain model

Proposed model

Identity-Based Encryption & Identity Management

  • Identity-Based Key Generator + OAuth/OpenID Connect
    Identity-Based Encryption - no passwords, no certificates, e-mail address
  • Identity provider + OAuth/OpenID Connect
    OAuth - open standard for authorization
    OpenID Connect - decentralized and secure authentication system
  • Client-side zero-knowledge encryption
    Cloud data storage provider has zero knowledge of the encryption keys
    Key generator has zero knowledge of users' data

Identity-Based Encryption is as strong as Identity Management itself!

Technology

Architecture based on standards

  • NIST SHA-256, AES-256, CTR-DRBG-256
  • OpenSSL FIPS 140-2 validated
  • OAuth 2.0 Identity Provider
  • OpenID Connect Provider

Pros

  • usability (no passwords, no certificates), lost key prevention
  • no certificates management (creation, storage, distribution, revocation)
  • IBE like features, key escrow/fair encryption, no need for receiver's public key before encryption
  • no IBE revocation problem (online service)

Cons

  • online service
  • master key security
